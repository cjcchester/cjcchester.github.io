Maven——有依赖每次打开项目仍然报红问题

发表于2023-12-22|更新于2023-12-22|其他工具Maven
|字数总计:1.1k|阅读时长:3分钟|阅读量:

如题，maven 项目 电脑每次重启打开 idea，依赖就全都找不到了，全部报红。
但是依赖包没道理莫名其妙消失，抱着这样的想法，开始解决问题了

  1. 尝试：

    1. 确保被 idea 识别为 maven 项目：idea，对应项目 pom.xml 右键是否有 maven？没有先 add maven 项目
    2. 项目结构看看 java 环境有没有什么问题
    3. 右键，settings.xml 看看有没有什么问题
    4. 设置 → 构建、执行、部署 → 构建工具 → maven，是否有问题
    5. 设置 → 构建、执行、部署 → 构建工具 → maven → 运行程序，是否有问题
    6. 设置 → maven → 仓库，更新 maven 索引（耗时）
    7. 项目 pom.xml 文件的目录下执行 mvn clean install -U 重下依赖包（耗时）
    8. 清理 idea 本地缓存重启（重启耗时）
    9. 如上每个操作都可以重启 idea，查看是否解决问题

  2. 解决：

    1. 一般来说，上面的就能解决，我的也可以，尤其是清理 idea 本地缓存

    2. 但是也没有任何道理！找了这么久都找不到问题！不应该这样

    3. 如下是 ai 的回答，问了这么久终于搞定了

    4. 我只是抱着尝试的心试了一下，终于，罪魁祸首找到了！问题就在 尝试禁用或卸载任何可能阻止 Idea 访问其缓存和配置文件的安全软件! 我什么也没做，在又一次报红时将 360 禁用了，重启 idea，惊喜发现，没有任何问题！依赖项全都好好的，尝试启动服务，也完全正常！ok，这下坚定了我删除 360 的决心了。

    5. 后续：删了 360 之后，深层原因找到了，如下图所示

      ​ 更多详见 Antivirus Impact on Build Speed，总的来说就是 防病毒软件影响构建 ，微软自带的还能提示一下，其他杀毒软件提示不了一点。但是都要

    6. 手动添加排除文件夹（为免麻烦，直接把 idea 的文件夹加进去，代码文件夹加进去）

    7. 关闭防护软件

    8. 关闭防护软件的实时保护、受控文件夹限制之类的

    9. ==后续==：删了 360，但是 windows 安全中心还是会导致一样的问题，而且我的很神奇，实时保护死活关不掉，关掉马上重启，这就很烦，试图注册表解决也不行，windows 家庭版也没组策略编辑器，干脆下了个火绒（因为 Windows 支持符合条件的杀毒软件存在后可以关闭 windows 防病毒），才发现火绒是真的小啊，安装包 22MB，比 360 不知道小到哪去，使用体验也很齐全了。尤其弹窗拦截十分牛逼，360 我要配半天设置才能把 360 自己的弹窗清完和其他设置弄好，wps 的弹窗它还拦不住……
